# リード獲得（フォーム）設定手順 — Formspree

サイトのフォーム（お問い合わせ／アドバイザー応募／求職者ウェイトリスト）は
**Formspree** に送信します。**現状はプレースホルダのため、実際には届きません。**
以下の手順で自分のフォームIDに差し替えてください（コード変更は不要、IDの置換のみ）。

## 仕組み

- 各 `<form>` は `assets/js/form.js` が処理し、`action="https://formspree.io/f/XXXX"` に POST します。
- 送信先が未設定（`YOUR_FORM_ID` のまま）の場合は、フォームに「送信先が未設定です」と表示され、誤って本番稼働しない安全設計です。

## セットアップ（10分）

1. [https://formspree.io](https://formspree.io) で無料アカウントを作成。
2. フォームを**用途ごとに**作成し、通知先メールを設定（推奨：3つ）。
   - 「お問い合わせ（トップ）」
   - 「アドバイザー応募（/join）」
   - 「求職者ウェイトリスト（/join）」
3. 各フォームの **エンドポイント**（`https://formspree.io/f/xxxxxxx`）を控える。
4. リポジトリ内の `YOUR_FORM_ID` を、対応するIDに置換する：

   | ファイル | フォーム | 置換箇所（`action` 属性） |
   |---|---|---|
   | `index.html` | お問い合わせ | `#contactForm` |
   | `join.html` | アドバイザー応募 | `#advisorForm` |
   | `join.html` | 求職者ウェイトリスト | `#waitlistForm` |

   一括検索：
   ```bash
   grep -rn "YOUR_FORM_ID" .
   ```
5. デプロイ（`main` にマージ）後、各フォームからテスト送信し、通知メールが届くことを確認。

## 補足

- **迷惑メール対策**：Formspree の管理画面で reCAPTCHA / honeypot を有効化できます。フォームには既に隠しフィールドを足す余地があります。
- **自動返信**：Formspree の「Autoresponse」で、応募者への自動返信メールを設定できます（丁寧な初動が信頼につながります）。
- **項目の増減**：フォームの入力欄（`name` 属性付き）はそのまま送信データになります。項目を足すときは `name` を付けるだけでOK。
- **将来の移行**：将来 Supabase 等の自社DBに切り替える場合も、`form.js` の送信先を差し替えるだけで対応できます（画面側の変更は不要）。

## プライバシーポリシー（要対応）

フォームの同意チェックはプライバシーポリシーへのリンク（現状 `href="#"`）を指します。
個人情報を取得するため、**公開前にプライバシーポリシーのページを用意**し、リンクを差し替えてください。

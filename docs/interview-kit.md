# インタビュー（CASE）取材キット — THE GATE JOURNAL

ハイクラス人材への体験談取材を、集客につながる記事にするための設計書です。
記事テンプレートは `templates/interview-template.html`、スタイルは `assets/css/article.css` の
`.case-summary / .interview-profile / .qa / .faq` を使用します。

> **そのまま取材に持ち込める詳細な質問票は [`docs/interview-questions.md`](./interview-questions.md) にあります。**
> 本ファイルはシリーズ設計・建付け・運用ルールをまとめた設計書です。

## 建付け（最初に必ず確認）

- 会社は**準備中**のため、扱うのは「**他社サービス利用時を含む、過去の転職体験**」の第三者取材です。
- **「MarkGateで成功した」と誤認させる表現はしない**（事実に反するため）。
- **匿名化＋本人同意**が前提。実名NGなら属性表記（例：`45歳・IT・経営企画`）、企業名は伏せる。
- 掲載前に**本人確認**を行い、**撤回可能**であることを伝える。数値は**本人申告に基づく**旨を注記。
- ステマ規制・景品表示法に配慮（対価を伴う場合は明示。基本は取材協力ベース）。

## 共通質問リスト（どの回でも聞く）

1. **転職前の状況** — 年代／役職／おおよその年収／当時の一番の不満は何でしたか。
2. **エージェントの選び方** — どうやって選びましたか。何社使いましたか。決め手は。
3. **担当者との関係** — 良かった点・悪かった点を**具体的なエピソード**で。「合わない」と感じた瞬間は。
4. **意思決定の分岐点** — 迷いは何でしたか。何が決め手で動きましたか（または動かなかった）。
5. **条件・年収交渉** — 提示額と最終額。交渉は誰がどう進めましたか。
6. **結果と“いま思うこと”** — 振り返って後悔・満足は。もう一度選べるなら何を変えますか。
7. **これから動く人へ** — 一番伝えたいことを一言で。

## 切り口別の深掘り

- **① 失敗系**：どこで「失敗だ」と気づいたか。どうリカバリーしたか。次に活かした教訓。
- **② 担当差**：前担当と後担当の違いを**数字・行動**で（提案の質、レスポンス、交渉の有無）。
- **③ 属性×成功**：45歳という年齢のリアル。捨てたもの・得たもの。周囲の反応。
- **④ 年収交渉**：提示額→最終額の推移。交渉のどの一手が効いたか。第三者の関与。
- **⑤ 踏みとどまり**：なぜ残したか。残って正解だったか。動くべき「時」の見極め。

## 記事化ワークフロー（取材後）

1. `templates/interview-template.html` を `media/<slug>.html` にコピー
2. `{{...}}` を置換し、**要点3つ（結論先出し）→プロフィール→リード→Q&A→まとめ→FAQ**を埋める
3. **FAQ本文と head の FAQPage JSON-LD を必ず一致**させる
4. `noindex` 行と `.sample-banner` を削除（公開可能に）
5. `media/index.html` に CASE カードを追加 ／ `sitemap.xml` に1行 ／ OG画像 `og-<slug>.png` を生成
6. 対応する**ノウハウ記事の末尾に「実例」として相互リンク**を追加（例：`advisor-gacha` → `case-agent-failure`）

## SEO要件（再掲）

- タイトルに属性＋悩みクエリ／冒頭で結論先出し／FAQ小見出しで長尾回収
- `Article` ＋ `FAQPage` 構造化データ／体験談↔ノウハウの双方向内部リンク
- URL・canonical・OGは既存と同じ規則（`.html`なし・末尾スラッシュなし・プレースホルダ域 `markgate.co.jp`）

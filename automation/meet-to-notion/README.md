# Google Meet → Notion 自動連携（Meet to Notion）

Google Meet の完了をきっかけに、Notion の **DB_Meeting** へ会議記録を自動で取り込む Google Apps Script です。

## 何が起きるか

Google Meet が終わると、Google 側が **Meet Recordings**（Google Drive）フォルダに次を自動保存します。

- 要約ドキュメント: `{会議名} - YYYY/MM/DD HH:MM JST - Gemini によるメモ`（概要・ネクストアクション・参加者・文字起こし）
- 録画: `{会議名} - … ～Recording`（mp4。会議で録画した場合のみ）

このスクリプトは **10分毎**にそのフォルダを見張り、新しい会議が見つかると Notion の DB_Meeting に1行作成して、次を埋めます。

| 反映先 | 内容 |
| --- | --- |
| `会議名` | 会議タイトル |
| `開催日時` | Google カレンダーの開始〜終了（取れない場合はファイル名の時刻） |
| `参加者` | カレンダーの参加者を Notion ユーザーに突き合わせて設定（未登録者は本文に記載） |
| `プロジェクト` | キーワード一致で DB_Project のプロジェクトに紐付け |
| `ファイル&メディア` | 要約Docを **.docx で実体アップロード** ＋ 録画は **Drive リンク** |
| 本文 | 概要／ネクストアクション（チェックボックス）／関連ファイル |

> 録画mp4は最大1GB級になり得るため Notion には複製せず Drive リンクで参照します（要約のみ実体アップロード）。

## セットアップ手順

### 1. Notion インテグレーションを作成し、DB に接続

1. <https://www.notion.so/my-integrations> で **内部インテグレーション**を作成（権限: 読み取り・更新・挿入）。
2. 表示される **Internal Integration Token**（`ntn_...`）を控える。
3. Notion で **DB**ページ（DB_Meeting と DB_Project の親）を開き、`···` → **コネクトを追加** から作成したインテグレーションを接続する。
   - DB_Meeting / DB_Project の両方にアクセスできればOK（親ページに接続すれば配下も有効）。

### 2. Apps Script プロジェクトを作成

1. <https://script.google.com> →「新しいプロジェクト」。
2. 既定の `コード.gs` を本リポジトリの [`Code.gs`](./Code.gs) の中身で置き換える。
3. （任意）`appsscript.json` を表示する設定（⚙️ → 「`appsscript.json` マニフェスト ファイルをエディタで表示する」）にして、本リポジトリの [`appsscript.json`](./appsscript.json) の内容に合わせる。

> このスクリプトは **あなたのアカウントとして** Drive / カレンダー / ドキュメントを読み取ります。Meet の録画・要約・カレンダーが入っている Google アカウントで作成してください。

### 3. スクリプト プロパティを設定

「プロジェクトの設定」⚙️ →「スクリプト プロパティ」で以下を追加します。

| プロパティ | 必須 | 値 |
| --- | --- | --- |
| `NOTION_TOKEN` | ✅ | 手順1で控えたトークン |
| `MEETING_DB_ID` |  | DB_Meeting のDB ID（既定: `1c955e9d58d68392aa61810b26f2e130`） |
| `PROJECT_DB_ID` |  | DB_Project のDB ID（既定: `37d55e9d58d680fb8beef32e8d4e22c4`） |
| `FOLDER_ID` |  | Meet Recordings フォルダID（既定: `1GRWj_BqR6KrwDvAHxTHR36kaCBHBxI29`） |
| `LOOKBACK_DAYS` |  | 遡及日数（既定: `7`） |
| `PROJECT_KEYWORDS` |  | プロジェクト紐付けのキーワード（任意・下記） |

既定値は現在のワークスペースに合わせてあるため、通常は **`NOTION_TOKEN` だけ**設定すれば動きます。

**`PROJECT_KEYWORDS`（任意）** — 会議内容からプロジェクトを当てる精度を上げたいときに、JSON で指定します。

```json
{
  "有料職業紹介免許の取得": ["職業紹介", "労働局", "募集情報", "免許", "職業安定法"],
  "会社登記": ["登記", "会社設立", "定款", "資本金", "会社名"]
}
```

未指定の場合はプロジェクト名そのものが本文中に出てくるかで判定します。

### 4. 初期セットアップを実行（トリガー設置）

1. 関数選択で **`setup`** を選び ▶ 実行。
2. 初回は OAuth 同意を求められるので許可（Drive 読み取り／カレンダー読み取り／ドキュメント読み取り／外部通信）。
3. 実行ログに Notion ユーザー一覧・プロジェクト一覧が出力され、**10分毎トリガー**が設置されます。

以降は Meet が終わるたび、最大10分以内に DB_Meeting へ自動反映されます。

## 動作確認・運用

- **手動テスト**: `runOnce` を実行（直近 `LOOKBACK_DAYS` 日の未処理会議を取り込み。既存分は重複ガードでスキップ）。
- **再取り込み**: `resetState` で処理済みフラグをクリア後 `runOnce`。
- **ログ**: Apps Script の「実行数」/ログで各会議の処理結果・エラーを確認できます。

### 重複防止

- 取り込み済みの要約Doc ID を記録（`PROCESSED_DOC_IDS`）。
- さらに作成前に Notion 側で **同名＋同日**の会議を検索し、存在すればスキップ。

## 仕組み（要点）

- **トリガー（＝Meet完了の代理）**: Meet 終了後に Drive へ要約/録画が保存されることを検知点にしています。Meet 単体には汎用的な「会議終了」通知が無いため、成果物（要約Doc）の出現を起点にするのが最も確実です。
- **参加者**: `参加者` は Notion の people 型のため、Notion ワークスペースに参加しているユーザーのみ設定できます。ゲスト（社外など）は本文の「参加者（Notion未登録）」に記載します。必要なら対象者を Notion ワークスペースに招待すると自動で列に入ります。
- **API**: Notion REST（バージョン `2022-06-28`）。ファイルは File Upload API（作成→送信→`file_upload` で添付）。

## 制限・注意

- 要約のアップロードは Notion の単一アップロード上限（数十MB）内を想定（Geminiメモは通常数KB〜MB）。録画は Drive リンク参照のため容量制限の影響を受けません。
- Notion へ共有された Drive リンクは、閲覧者の Drive 権限に従います（社外共有時は Drive 側の共有範囲を確認してください）。
- 文字起こしが極端に短い回（要約が生成されない回）でも行は作成され、本文は取得できた範囲のみになります。
